# Hosted JSON API

This document describes the frontend-ready schedule JSON generated by the
standalone parser pipeline in:

```text
/Users/gaojiahao/Documents/GitHub/tools
```

The mobile app consumes these files as static JSON. There is no runtime backend
API and no Supabase Edge Function involved in app reads.

## Entry Point

The stable public entry point is:

```text
https://<project-ref>.supabase.co/storage/v1/object/public/planner-data/data/mobile/v1/manifest.json
```

Resolve all file references in `manifest.json` relative to:

```text
https://<project-ref>.supabase.co/storage/v1/object/public/planner-data/data/mobile/v1/
```

Recommended refresh flow:

1. Fetch `manifest.json`.
2. Compare `manifest.revision` with the locally cached revision.
3. If the revision changed, fetch `seasonFile`, `carsFile`, and `tracksFile`.
4. Cache the manifest and all referenced files together.
5. Continue using cached data if refresh fails.

## Manifest

Example:

```json
{
  "schemaVersion": 1,
  "seasonId": "2026-s3",
  "revision": "b973e3f6",
  "generatedAt": "2026-06-26T15:05:47Z",
  "seasonFile": "releases/2026-s3/b973e3f6/season.json",
  "carsFile": "releases/2026-s3/b973e3f6/cars.json",
  "tracksFile": "releases/2026-s3/b973e3f6/tracks.json",
  "checksums": {
    "releases/2026-s3/b973e3f6/season.json": "sha256:...",
    "releases/2026-s3/b973e3f6/cars.json": "sha256:...",
    "releases/2026-s3/b973e3f6/tracks.json": "sha256:..."
  }
}
```

Fields:

| Field | Type | Notes |
| --- | --- | --- |
| `schemaVersion` | Integer | Current value is `1`. |
| `seasonId` | String | Season identifier, for example `2026-s3`. |
| `revision` | String | Content revision. Use this for cache invalidation. |
| `generatedAt` | String | UTC timestamp for the generated release. |
| `seasonFile` | String | Relative path to `season.json`. |
| `carsFile` | String | Relative path to `cars.json`. |
| `tracksFile` | String | Relative path to `tracks.json`. |
| `checksums` | Object | Optional client-side integrity map keyed by relative file path. |

## Season File

Top-level shape:

```json
{
  "schemaVersion": 1,
  "generatedAt": "2026-06-26T15:05:47Z",
  "seasonId": "2026-s3",
  "seasonName": "2026 Season 3",
  "seasonStart": "2026-06-16T00:00:00Z",
  "seasonEnd": "2026-09-08T00:00:00Z",
  "weekSeasonStart": "2026-06-16T00:00:00Z",
  "weeks": [],
  "series": [],
  "races": []
}
```

Use `races[]` as the primary source for schedule list screens. `series[]`
contains metadata and `raceIds[]` for grouping or series detail screens.

### `weeks[]`

```json
{
  "weekNumber": 1,
  "startsAt": "2026-06-16T00:00:00Z",
  "endsAt": "2026-06-23T00:00:00Z"
}
```

### `series[]`

```json
{
  "seriesId": "series_advanced_mazda_mx_5_cup_by_heusinkveld_2026_s3",
  "name": "Advanced Mazda MX-5 Cup by Heusinkveld",
  "category": "Sports Car",
  "license": {
    "className": "Class D",
    "safetyRating": 4.0,
    "raw": "Class D 4.0 --> Pro/WC 4.0"
  },
  "isOfficial": true,
  "setupType": "unknown",
  "setupSource": "unknown",
  "startType": "standing",
  "startTypeSource": "weatherText",
  "raceIds": [
    "2026-s3-series_advanced_mazda_mx_5_cup_by_heusinkveld_2026_s3-w1-track_n_rburgring_nordschleife_industriefahrten"
  ]
}
```

Series enums:

| Field | Values |
| --- | --- |
| `setupType` | `fixed`, `open`, `unknown` |
| `startType` | `rolling`, `standing`, `unknown` |

`setupSource` and `startTypeSource` explain where the parser found the value.
The frontend may display `unknown` as an unset/manual-review value.

### `races[]`

```json
{
  "raceId": "2026-s3-series_advanced_mazda_mx_5_cup_by_heusinkveld_2026_s3-w1-track_n_rburgring_nordschleife_industriefahrten",
  "seriesId": "series_advanced_mazda_mx_5_cup_by_heusinkveld_2026_s3",
  "seriesName": "Advanced Mazda MX-5 Cup by Heusinkveld",
  "category": "Sports Car",
  "weekNumber": 1,
  "startsAt": "2026-06-16T00:00:00Z",
  "endsAt": "2026-06-23T00:00:00Z",
  "trackPackageId": "track_n_rburgring_nordschleife_industriefahrten",
  "trackName": "Nürburgring Nordschleife",
  "trackConfigName": "Industriefahrten",
  "carSkus": ["car_global_mazda_mx_5_cup"],
  "carClasses": ["Global Mazda MX 5 CUP"],
  "setupType": "unknown",
  "setupSource": "unknown",
  "startType": "standing",
  "startTypeSource": "weatherText",
  "raceLength": {
    "laps": 5
  },
  "precipChance": 0,
  "sessions": [
    {
      "type": "recurring",
      "firstSessionOffsetMinutes": 15,
      "repeatEveryMinutes": 60
    }
  ]
}
```

Race fields:

| Field | Type | Notes |
| --- | --- | --- |
| `raceId` | String | Stable race row ID. |
| `seriesId` | String | Links to `series[].seriesId`. |
| `seriesName` | String | Denormalized display name for list rows. |
| `category` | String | Display/filter category. |
| `weekNumber` | Integer | Links to `weeks[].weekNumber`. |
| `startsAt` / `endsAt` | String | UTC week window for this race. |
| `trackPackageId` | String | Links to `tracks[].packageId`. |
| `trackName` | String | Display track name. |
| `trackConfigName` | String | Optional layout/config display name. |
| `carSkus` | Array | Links to `cars[].sku`. |
| `carClasses` | Array | Display class labels. |
| `setupType` | String | `fixed`, `open`, or `unknown`. |
| `startType` | String | `rolling`, `standing`, or `unknown`. |
| `raceLength` | Object | Optional, either laps or minutes. |
| `precipChance` | Number | Optional rain chance percentage. |
| `sessions` | Array | Session timing rules. |

`raceLength` is one of:

```json
{ "laps": 15 }
```

```json
{ "minutes": 20 }
```

`sessions[]` uses one of these shapes:

```json
{
  "type": "recurring",
  "firstSessionOffsetMinutes": 15,
  "repeatEveryMinutes": 60
}
```

```json
{
  "type": "setTimes",
  "offsetMinutes": [0, 1440, 2880]
}
```

`offsetMinutes` are relative to `race.startsAt`.

## Cars File

Top-level shape:

```json
{
  "schemaVersion": 1,
  "generatedAt": "2026-06-26T15:05:47Z",
  "cars": []
}
```

Entry shape:

```json
{
  "sku": "car_global_mazda_mx_5_cup",
  "displayName": "Global Mazda MX 5 CUP",
  "categories": ["Sports Car"],
  "carClasses": ["Global Mazda MX 5 CUP"]
}
```

Use `sku` to join from `race.carSkus`.

## Tracks File

Top-level shape:

```json
{
  "schemaVersion": 1,
  "generatedAt": "2026-06-26T15:05:47Z",
  "tracks": []
}
```

Entry shape:

```json
{
  "packageId": "track_adelaide_street_circuit",
  "displayName": "Adelaide Street Circuit",
  "type": "road",
  "supportedTypes": ["road"],
  "sourceTrackIds": []
}
```

Use `packageId` to join from `race.trackPackageId`.

Known track `type` values:

```text
road
oval
dirtOval
dirtRoad
unranked
```

## Recommended Frontend Mapping

Build lookup maps after loading all files:

```kotlin
val carsBySku = cars.associateBy { it.sku }
val tracksByPackageId = tracks.associateBy { it.packageId }
val seriesById = season.series.associateBy { it.seriesId }
```

Then each schedule row can be built from:

```text
race
  -> seriesById[race.seriesId]
  -> tracksByPackageId[race.trackPackageId]
  -> race.carSkus.mapNotNull(carsBySku::get)
```

Do not reconstruct race rows from series metadata. The generated
`season.races[]` array is already the frontend-ready schedule list.

## Preference Data

Hosted JSON does not contain user-specific data. Keep these local-only:

- owned cars
- owned tracks
- favorite cars
- favorite tracks
- account identifiers
- credentials
